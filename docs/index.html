Introduction #

AssetIT is a comprehensive Laravel 13 application designed to help IT departments manage their hardware assets, software licenses, employees, vendors, and maintenance records efficiently.

Built with modern web technologies including Bootstrap 4, Chart.js, and the Spatie Permission package, AssetIT provides a robust foundation for IT asset management with role-based access control, detailed reporting, and automated license expiration alerts.

100% Free & Open Source Forever! This project is licensed under MIT. Use it, modify it, and distribute it as you wish - completely free, forever.
Quick Info: This application includes 120+ seeded hardware assets, 50 software licenses, 30 employees, and 80 maintenance records for demonstration purposes.

Features #

AssetIT comes packed with powerful features for complete IT asset management.

Hardware Asset Management

Track computers, servers, monitors, and peripherals with full CRUD operations and detailed specifications.

Software License Management

Manage software licenses with seat tracking, expiration dates, and automatic renewal alerts.

Employee Management

Maintain employee directory with department assignments and contact information.

Department Management

Organize employees by department with hierarchical structure support.

Vendor Management

Manage vendor relationships, contacts, and purchase information.

Maintenance Records

Track hardware maintenance, repairs, costs, and service history.

Administration Features

Role-Based Access Control

Four predefined roles: Admin, IT Manager, Technician, and Viewer with granular permission management.

Audit Logging

Complete audit trail of all system changes for compliance and tracking.

Reports & Analytics

Dashboard with Chart.js visualizations, PDF and Excel export capabilities.

Automated Alerts

Scheduled license expiration checks with email notifications.

Multi-Language Support

Available in English and French.

Installation #

Prerequisites

  • PHP 8.4 or higher
  • Composer
  • MySQL 5.7+ or MariaDB
  • Node.js & npm
  • Web Server (Apache/Nginx)

Step 1: Clone Repository

git clone https://github.com/brahbassim/assetit.git
cd assetit

Step 2: Install Dependencies

composer install
npm install

Step 3: Environment Configuration

cp .env.example .env

Edit the .env file and configure your database credentials:

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=assetit
DB_USERNAME=root
DB_PASSWORD=your_password

Step 4: Generate Application Key

php artisan key:generate

Step 5: Run Migrations & Seed Data

php artisan migrate --seed

Step 6: Build Assets

npm run build
Done! Your application is now set up. Continue to the next section to start the server.

Configuration #

Mail Configuration

Configure mail settings for license expiration alerts:

MAIL_MAILER=smtp
MAIL_HOST=smtp.mailtrap.io
MAIL_PORT=2525
MAIL_USERNAME=null
MAIL_PASSWORD=null
MAIL_ENCRYPTION=tls
MAIL_FROM_ADDRESS="noreply@assetit.com"
MAIL_FROM_NAME="${APP_NAME}"

Scheduler Configuration

Add the scheduler cron job to check for expiring licenses daily:

# Add to crontab
* * * * * php /path-to-project/artisan schedule:run >> /dev/null 2>&1

Queue Configuration

For processing background jobs:

QUEUE_CONNECTION=database

Cache Configuration

CACHE_STORE=file
SESSION_DRIVER=file

User Roles #

Admin

Full access to all features including:

  • All CRUD operations
  • User management
  • Role management
  • Permission management
  • All reports
  • Audit logs
IT Manager

Manage assets and view reports:

  • All CRUD operations
  • All reports
  • Employee management
  • No user/role management
Technician

Hardware-focused access:

  • Hardware assets CRUD
  • Maintenance records CRUD
  • View employees
  • Basic reports
Viewer

Read-only access:

  • View all data
  • View reports
  • No modifications

Permission Categories

The system includes 35+ granular permissions organized by module:

  • Hardware Assets: view, create, edit, delete
  • Software Licenses: view, create, edit, delete
  • License Assignments: view, create, edit, delete
  • Employees: view, create, edit, delete
  • Departments: view, create, edit, delete
  • Vendors: view, create, edit, delete
  • Asset Categories: view, create, edit, delete
  • Maintenance Records: view, create, edit, delete
  • Reports: view, export
  • Users: view, create, edit, delete
  • Roles: view, create, edit, delete, assign
  • Permissions: view, manage

API Documentation #

Note: The current version uses web routes with Blade templates. REST API endpoints can be added as needed.

Adding API Support

To add REST API support, install Laravel Sanctum:

composer require laravel/sanctum
php artisan vendor:publish --provider="Laravel\Sanctum\SanctumServiceProvider"
php artisan migrate

Example API Routes

After installing Sanctum, you can define API routes in routes/api.php:

Route::middleware('auth:sanctum')->group(function () {
    Route::apiResource('assets', HardwareAssetController::class);
    Route::apiResource('licenses', SoftwareLicenseController::class);
    Route::get('/dashboard/stats', [DashboardController::class, 'stats']);
});

Deployment #

Production Checklist

  1. Set APP_ENV=production in .env
  2. Set APP_DEBUG=false in .env
  3. Configure secure database credentials
  4. Set up HTTPS/SSL
  5. Configure mail driver

Optimize for Production

php artisan config:cache
php artisan route:cache
php artisan view:cache
php artisan optimize

Shared Hosting

For shared hosting environments:

php artisan config:cache
php artisan route:cache

Ensure the storage and bootstrap/cache directories are writable.

Docker Deployment

Create a Dockerfile in your project root:

FROM php:8.4-fpm

WORKDIR /var/www

COPY composer.json composer.lock ./
RUN composer install --no-dev --optimize-autoloader

COPY . .
RUN php artisan config:cache && \
    php artisan route:cache && \
    php artisan view:cache

EXPOSE 9000
CMD ["php-fpm"]
